## 2. Bugs & Issues

### 2.1 CRITICAL - Data Inconsistency στο Dashboard 🔴
**Περιγραφή**: Ασυμφωνία μεταξύ Dashboard stats και πραγματικών δεδομένων κτιρίου.

**Αναπαραγωγή**:
1. Login ως Admin
2. Στο Dashboard βλέπω: "1 κτίριο", "0,00€ Οφειλές"
3. Στο widget "Κατάσταση Κτιρίων" βλέπω: 2 κτίρια
4. Βουλής 6 δείχνει: 7.090€ υπόλοιπο, 210€ οφειλές

**Αναμενόμενο**: Τα stats να συμφωνούν με τα δεδομένα των κτιρίων.

**Πιθανή αιτία**: Τα stats πιθανώς υπολογίζονται μόνο για το "επιλεγμένο" κτίριο αντί για όλα.

**Προτεραιότητα**: HIGH

---

### 2.2 MINOR - Date Format Inconsistency ⚠️
**Περιγραφή**: Στη φόρμα δαπάνης, η ημερομηνία εμφανίζεται σε ISO format (2025-11-29) αντί για ελληνικό format (29/11/2025).

**Εντοπισμός**: Financial → Δαπάνες → Νέα Δαπάνη → Πεδίο "Ημερομηνία"

**Αναμενόμενο**: Ελληνικό date format για consistency με υπόλοιπη εφαρμογή.

**Προτεραιότητα**: LOW

---

### 2.3 MINOR - Typo in Building Name ⚠️
**Περιγραφή**: "Βουλής 6 -Demo" έχει περίεργο spacing πριν το "-Demo".

**Προτεραιότητα**: LOW

## 3. Θετικά Σημεία / Best Practices

### 3.1 Εξαιρετικά UX Patterns
1. **Smart Form Completion**: Η φόρμα δαπάνης αυτόματα συμπληρώνει πεδία βάσει κατηγορίας
2. **Real-time Updates**: Το sidebar ενημερώνεται αυτόματα μετά από αλλαγές
3. **Empty States**: Σαφή μηνύματα όταν δεν υπάρχουν δεδομένα με call-to-action
4. **Filter Badges**: Εμφάνιση ενεργών φίλτρων με tags
5. **Building Selector Modal**: Εξαιρετική οργάνωση με "Τρέχον" tag

### 3.2 Accessibility
- Καλή χρήση headings (h1, h2, h3)
- Icons με κείμενο (όχι μόνο icons)
- Keyboard navigation (δοκιμάστηκε βασικά)

### 3.3 Performance
- Login: ~3 sec
- Page navigation: ~2 sec
- Data refresh: Γρήγορο
